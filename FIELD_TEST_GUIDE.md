# AI Scene Director RC16 현장 사용성·설치 점검

## 목적

첫 사용자가 장면 생성부터 수정·출력·복원까지 완료할 수 있는지와 플랫폼 설치 앱이 실제 제작 흐름을 유지하는지 확인합니다.

## 권장 참여자

AI 이미지·영상 제작 경험이 있지만 Blender·Maya 전문가는 아닌 제작자 3~5명.

## 제작 과제

1. 명령 검색에서 `AI 장면 초안 만들기`를 실행합니다.
2. 두 인물이 카페에서 대화하는 장면을 만듭니다.
3. 주인공의 위치를 바꿉니다.
4. 두 번째 Shot의 카메라 구도와 FOV를 수정합니다.
5. 인물에게 걷기 또는 뒤돌아보기 Action을 추가합니다.
6. 프로젝트 점검에서 차단 문제가 없는지 확인합니다.
7. 현재 AI용 내보내기를 출력합니다.
8. 프로젝트 번들을 저장한 뒤 다시 불러옵니다.

## 관찰 항목

- 명령 검색을 스스로 발견하는가
- 첫 수정 대상과 이동 핸들을 이해하는가
- Scene, Shot, Action의 차이를 설명할 수 있는가
- 출력 차단 문제를 스스로 해결하는가
- 프로젝트 번들과 AI용 내보내기를 구분하는가
- 15분 안에 과제를 완료하는가

## 세션 기록

앱의 `세션 기록`에서 익명 JSON을 내려받습니다. 프롬프트, 장면명, 파일명, 서버 주소는 포함되지 않습니다.

별도 기록:

- 완료 여부
- 도움을 요청한 지점
- 잘못 누른 버튼
- 이해하기 어려웠던 용어
- 가장 유용했던 기능
- 숨겨도 된다고 느낀 기능

## 설치 검증

각 운영체제에서 다음을 확인합니다.

- 설치 파일 실행과 첫 시작
- WebGL 뷰포트 또는 안전 모드 표시
- 프로젝트 폴더 연결
- 앱 종료 후 복구본 확인
- GLB 한 개 가져오기
- 프로젝트 번들 저장·재실행·복원
- AI용 내보내기 출력
- 제거 프로그램 또는 앱 삭제

실패 시 `프로젝트 점검 → 지원 번들 ZIP`과 해당 플랫폼의 `BROWSER_SMOKE`, `NATIVE_ARTIFACTS`, `NATIVE_RUNTIME` 보고서를 함께 수집합니다.


## RC16 뷰포트·조명 현장 점검

1. 어두운 프리셋에서 `작업 밝기` 켜기/끄기를 비교합니다.
2. 작업 밝기를 켠 상태에서 기준 이미지를 내보내고, 편집용 보조광이 결과에 섞이지 않는지 확인합니다.
3. 필 라이트를 선택해 X·Y·Z 범위 링이 보이는지, 각도 입력은 비활성화되는지 확인합니다.
4. 방향광을 회전해 평행 화살표와 실제 조명 방향이 함께 바뀌는지 확인합니다.
5. 스포트라이트를 선택해 범위와 각도 변경이 조사 콘에 반영되는지 확인합니다.
6. 선택한 조명 구체를 직접 드래그해 카메라는 고정되고 조명 위치만 바뀌는지 확인합니다.
7. X/Y/Z 이동 핸들로 축별 정밀 이동이 가능한지 확인합니다.
8. 조작 도움말을 접고 페이지를 다시 열어 접힌 상태가 유지되는지 확인합니다.

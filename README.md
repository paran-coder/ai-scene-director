# AI Scene Director MVP 0.8

외부 유료 추론 API 없이 자연어로 다중 인물·소품·카메라·샷을 구성하고, 3D에서 수정해 Shot Package 또는 로컬 ComfyUI로 전달하는 브라우저 기반 프로토타입입니다.

## 실행

```bash
npm install
npm run dev
```

자동 테스트와 프로덕션 빌드:

```bash
npm test
npm run build
npm run preview
```

## 0.8 핵심 기능

### 로컬 자연어 씬 생성

상단의 **AI 씬 생성** 버튼에서 한 문장 또는 여러 문장으로 장면을 설명할 수 있습니다.

예:

```text
비 오는 밤의 편의점 앞이다. 검은 코트를 입은 여성과 교복을 입은 남학생이 마주 보고 있다.
여성은 우산을 들고 있고 남학생은 자전거 옆에 서 있다.
처음에는 두 사람이 함께 보이는 와이드 샷, 다음은 여성의 얼굴 클로즈업,
마지막에는 남학생이 자전거를 타고 떠나는 트래킹 샷으로 만들어줘.
```

규칙 기반 로컬 컴파일러가 다음을 생성합니다.

- 등장인물과 설명
- 소품과 개수
- 장소·시간대·분위기 태그
- 인물 배치
- 바라보기·손에 들기·앉기·표면 배치 관계
- 와이드·미디엄·클로즈업·오버숄더·인서트·트래킹·로우/하이 앵글 샷
- 샷별 카메라 Entity와 방향
- 걷기·뒤돌아보기·카메라 돌리 인 Action
- 두 개의 기본 조명

적용 전 분석 결과를 카드 형태로 확인할 수 있습니다. 적용하면 현재 Scene 전체가 하나의 `replaceScene` Transaction으로 교체되며 실행 취소할 수 있습니다.

### 기존 기능

- Scene 기본 상태와 Shot Override
- 다중 객체 편집과 샷 관리
- 휴머노이드 관절·포즈 프리셋·팔 IK
- 바라보기·손에 들기·앉기·표면 배치 관계
- Action Block과 기본 타임라인
- 샷 카메라 보기
- 시작·종료 프레임, Pose·Depth·Mask Shot Package
- 로컬 ComfyUI 선택 연결과 결과 Shot 등록
- JSON 프로젝트 내보내기·불러오기
- 프로젝트 스키마 0.8.0 마이그레이션

## 자연어 컴파일러의 현재 인식 범위

- 인물: 여성, 남성, 학생, 소년·소녀, 경찰, 직원, 손님, 의사 등
- 고유 이름: 문장 시작의 `지윤과 민수가 ...` 형태
- 장소: 편의점, 카페, 거리, 방, 사무실, 옥상, 숲, 해변 등
- 소품: 테이블, 의자, 컵, 우산, 자전거, 가방, 소파, 자동차 등
- 샷: 와이드, 미디엄, 클로즈업, 오버숄더, 인서트, 트래킹, 로우·하이 앵글
- 행동·관계: 마주 보기, 바라보기, 들기, 앉기, 떠나기, 뒤돌기, 돌리 인, 오빗

명시된 샷이 없으면 와이드 → 미디엄 → 클로즈업의 기본 3샷을 제안합니다.

## ComfyUI 연결

ComfyUI는 고급 사용자용 선택 어댑터입니다. 기본 주소는 `http://127.0.0.1:8188`입니다.

브라우저에서 CORS 오류가 발생하면 ComfyUI를 앱의 개발 주소에 맞는 CORS 허용 옵션과 함께 실행해야 할 수 있습니다.

```bash
python main.py --enable-cors-header http://localhost:5173
```

## 제한 사항

- 자연어 해석은 로컬 규칙 기반이며 자유로운 문학적 문장 전체를 이해하는 LLM은 아닙니다.
- 배경은 현재 Scene 이름과 설명으로 저장되며 실제 3D 환경 에셋 자동 배치는 아직 제한적입니다.
- 의상·외형 설명은 캐릭터 이름에 보존되지만 실제 캐릭터 외형에 시각적으로 적용되지는 않습니다.
- 일반 소품은 기본 박스 프록시로 표시됩니다.
- 카메라 렌즈·FOV는 아직 Shot 유형별 전용 메타데이터로 저장하지 않습니다.
- ComfyUI 자체와 모델·커스텀 노드는 사용자가 별도로 설치해야 합니다.
- JavaScript 번들은 아직 크므로 이후 코드 분할이 필요합니다.

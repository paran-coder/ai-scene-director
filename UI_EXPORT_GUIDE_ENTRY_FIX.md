# 내보내기 사용법 헤더 버튼 표시 수정

## 원인

기존 헤더 버튼은 화면의 CSS 너비가 1320px 이하가 되면 `내보내기 사용법` 글자를 숨기고 `?` 아이콘만 남겼습니다. 브라우저 확대가 적용되거나 창이 좁은 환경에서는 버튼이 없는 것처럼 보일 수 있었습니다.

기존 브라우저 검사는 DOM에서 버튼을 클릭할 수 있는지만 확인했으며, 실제 화면에서 글자가 보이는지와 버튼 너비는 검사하지 않았습니다.

## 수정

- `AI용 내보내기` 바로 오른쪽에 사용법 버튼 유지
- 일반 화면에서는 `? 내보내기 사용법`을 항상 표시
- 좁은 화면에서도 아이콘만 남기지 않고 `? 사용법` 표시
- 데스크톱 최소 너비 128px, 축약 상태 최소 너비 78px 적용
- 노란색 테두리와 옅은 배경으로 독립 버튼임을 강화
- 앱과 프로젝트 스키마 버전을 `1.0.0-rc.12`로 갱신

## 회귀 검사

브라우저 검사는 다음을 실제 계산 좌표와 스타일로 확인합니다.

- 버튼 요소 존재
- `display`, `visibility`, `opacity` 기준으로 화면 표시
- 버튼 너비 70px 이상, 높이 30px 이상
- 텍스트에 `사용법` 포함
- 텍스트 라벨이 `display:none`이 아님
- `AI용 내보내기` 버튼 오른쪽 12px 이내에 배치
- 버튼 클릭 후 전체 사용법 페이지 열림

1440×920과 1366×768 프로필에서 모두 통과해야 배포 검사가 성공합니다.
